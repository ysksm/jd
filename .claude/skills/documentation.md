## 更新が必要なケース

### README.md の更新が必要

1. **新機能追加時**
   - 「特徴」セクションに追加
   - 必要に応じて「コマンドリファレンス」を更新

2. **CLIコマンド変更時**
   - 「コマンドリファレンス」セクション
   - 「使い方」セクションの例

3. **依存関係変更時**
   - 「前提条件」セクション

4. **設定フォーマット変更時**
   - 設定ファイル例を更新

5. **データベーススキーマ変更時**
   - 「データベーススキーマ」セクション

### CLAUDE.md の更新が必要

1. **アーキテクチャ変更時**
   - モジュール構成、レイヤー構造

2. **API変更時**
   - JIRA API統合、エンドポイント

3. **データベーススキーマ変更時**
   - テーブル定義、インデックス

4. **新しいuse case追加時**
   - use_cases一覧

5. **MCP ツール追加時**
   - MCPツール一覧

6. **開発コマンド変更時**
   - Development Commands セクション

### docs/ARCHITECTURE.md の更新が必要

1. **新しいcrate追加時**
2. **レイヤー構造変更時**
3. **新しいrepository追加時**
4. **新しいentity追加時**

### docs/EMBEDDINGS.md の更新が必要

1. **新しい埋め込みプロバイダー追加時**
2. **埋め込みモデル変更時**
3. **埋め込み設定変更時**

### docs/MCP.md の更新が必要

1. **MCPツール追加/削除時**
2. **ツールパラメータ変更時**
3. **トランスポート変更時**

## ドキュメント記述スタイル

### README.md
- ユーザー向け、分かりやすく
- 日本語で記述
- 具体的なコマンド例を含める
- 絵文字を使用してセクションを視覚的に区別

### CLAUDE.md
- 開発者/AI向け、技術的に正確に
- 日本語/英語混在可
- コード例を含める
- アーキテクチャ決定の理由を記載

### docs/ 以下
- 詳細な技術ドキュメント
- 日本語で記述
- ダイアグラムやテーブルを活用

## 注意事項

- ドキュメントの更新を忘れると、コードと文書の乖離が発生
- 特に README.md はユーザーが最初に見るため、常に最新を維持
- CLAUDE.md は AI が参照するため、正確性が重要
- スキーマ変更時は必ず `.claude/skills/database-schema.md` も更新
